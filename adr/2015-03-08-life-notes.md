# Заметки о поездках

## Контекст

Заметки о поездках хранятся в Blade-шаблонах на диске в `resources/views/life/trips/`. Каждая поездка — отдельный файл. Такой подход позволяет удобно работать с заметками локально, делать массовые правки и использовать Git для версионирования.

## Решение

### Рабочий процесс

Написание заметки проходит в несколько этапов:

**1. Начальный черновик** — только список фотографий, без текста. Файл содержит bare-имена изображений, по одному на строку. Фотографии ещё не загружены на сервер.

```blade
@extends('life.trips.base')

@section('content')
IMG_0001.jpeg
IMG_0002.jpeg
IMG_0003.jpeg
@endsection
```

**2. Первый черновик** — добавляется текст описаний. Используется `@ru/@en/@endru` для локализации. Изображения остаются bare-именами как якоря. Английский перевод опционален.

```blade
@ru
  <p>Вегетарианское кафе «Джаганнат».</p>
@en
  <p>Jagannat vegetarian cafe.</p>
@endru
IMG_0001.jpeg
```

**3. Редактирование** — перечитывание, удаление неудачных фото, объединение нескольких снимков под общим описанием.

**4. Загрузка фотографий** — обработка и загрузка на удалённый сервер. После этого изображения доступны по прямым ссылкам.

**5. Финализация** — запуск скрипта, который оборачивает bare-имена в `@include` директивы:

```blade
@include('tpl.pic-2x', ['pic' => 'IMG_1675.jpg'])
@include('tpl.fotorama-2x', ['pics' => [
  'IMG_1665.jpg',
  'IMG_1667.jpg',
  'IMG_1668.jpg',
]])
```

**6. Публикация** — финальное прочтение с отображаемыми изображениями, коммит, публикация.

### Почему bare-имена не рендерятся

Bare-имена файлов в черновиках намеренно выводятся как обычный текст, а не преобразуются в `<img>` теги. Это обусловлено рабочим процессом:

- Черновики пишутся до загрузки фотографий на сервер, потому что обрабатывать фотографии долго и заранее неизвестно найдется ли что сказать о фотографии
- Автоматический рендеринг привёл бы к 404 для отсутствующих изображений
- Bare-имена служат визуальными якорями — видно, какие фото будут в заметке

### Шаблоны изображений

Существующие `tpl.*` шаблоны:

| Шаблон | Назначение |
|---|---|
| `tpl.pic` | Одиночное изображение, 1x (без srcset) |
| `tpl.pic-2x` | Одиночное изображение, 2x с srcset |
| `tpl.pic-arbitrary` | Одиночное изображение произвольного размера, 1x |
| `tpl.pic-arbitrary-2x` | Одиночное изображение произвольного размера, 2x |
| `tpl.fotorama` | Галерея из нескольких изображений, 1x |
| `tpl.fotorama-2x` | Галерея из нескольких изображений, 2x |

Качество 1x использовалось для давних снимков с маленьким разрешением. После появления относительно современной камеры, все снимки публикуются в качестве 2x.

2x-шаблоны используют srcset для адаптивной подачи изображений (1000w/2000w) и отображают оверлей-иконки (маркер карты, ссылка на оригинал).

### Локализация

Для сокращения синтаксиса локализации были созданы новые директивы blade:

```php
Blade::directive('ru', fn () => '<?php if ($locale === "ru"): ?>');
Blade::directive('en', fn () => '<?php elseif ($locale === "en"): ?>');
Blade::directive('endru', fn () => '<?php endif; ?>');
```

Упор был сделан на однократную передачу текущей локализации в шаблон, потому что в тексте заметки может быть несколько сотен локализованных блоков.

Русский текст в заметке обязателен, английский — опционален.

## Последствия

### Положительные

- Файлы на диске — удобно для локальной работы, массовых правок, Git
- Bare-имена как якоря — не ломают черновики отсутствующими 404
- Разделение этапов черновика и финализации — можно писать текст без ожидания загрузки фото
- Переиспользуемые `tpl.*` шаблоны — единый стиль всех изображений

### Отрицательные

- Многословный синтаксис `@include` для вставки изображений
- Необходимость запускать скрипт финализации вручную
